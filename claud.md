## 開発の基本ルール
1. **既存コードの尊重**:
   - `space.md` に記載されたアーキテクチャやルールを遵守してください。
   - `output: 'export'` (静的ビルド) を前提としているため、SSRやAPI Routesを使用する実装は避けてください。
2. **Firestoreの操作**:
   - データの取得・更新は `lib/firebase.ts` で初期化されたインスタンスを利用し、クライアントサイドで行ってください。
   - 既存のデータ構造（`year_YYYY` や `year_YYYY_termX_tasks`）を壊さないよう、マージ処理に注意してください。
3. **コードの品質**:
   - TypeScriptの型定義を適切に行い、`any`の利用は最小限に抑えてください。
   - UIの変更には Tailwind CSS を使用し、既存のデザインテイスト（色使い、余白など）に合わせるよう努めてください。

## 🚀 現在の主要な改修タスク
### タスク: クライアント用画面 (`app/client/page.tsx`) のFirestore連携と本番実装

**背景**:
現在、クライアント用画面はモックアップ状態であり、ハードコードされたデータと状態（`useState`）のみで動いています。これをFirestoreと連携させ、管理画面(`dashboard/detail`)とデータが同期するように改修する必要があります。

**具体的な実装要件**:
1. **URLからのパラメータ取得と認証の代替**:
   - 顧問先はログインなしでアクセスするため、URLパラメータ (例: `?id=XXX&year=YYYY&term=Z`) から対象のドキュメントを特定する処理を実装してください。
2. **データの取得 (Read)**:
   - Firestoreの `clients` コレクションから該当する顧問先データを取得し、画面に初期表示してください。
   - `dashboard/detail` で定義されているタスク一覧（`INITIAL_TASKS`など）と同期する仕組みを構築してください。
3. **データの更新・保存 (Update)**:
   - タスクのチェックボックス操作や、売上・仕入などの数値入力が行われた際、オートセーブ（数秒遅れの自動保存）でFirestoreの該当データを更新するロジックを実装してください。
   - 数値の突合ロジック（10%以内の誤差判定等）を正しく動作させてください。
4. **提出処理 (Submit)**:
   - すべてのタスクが完了し、誤差判定の確認（または理由の入力）が済んだ場合のみ、「提出する」ボタンを有効化してください。
   - 提出ボタン押下時に、Firestore上の `clientStatus` を「完了」に更新し、タイムスタンプ (`completedAt`) を記録してください。
